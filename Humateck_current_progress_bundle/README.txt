============================================================
⚠️ 절대 금지 지침 / 다음 담당자 필독
------------------------------------------------------------
이 시스템은 배송 시스템이다. 번역기, 검수기, 품질판단기, 거부기가 아니다.
사용자 명시 승인 없이는 아래 항목을 절대 추가, 복구, 변경하지 말 것.

[절대 금지]
1) 검수 로직 추가 금지
2) 자체 판단 로직 추가 금지
3) 자체 거부 / 반려 / 차단 / 보류 / 감점 / 제외 로직 추가 금지
4) 형식검사 실패 문구, 번역문 점검 문구, 자체 오류판정 문구 추가 금지
5) 불필요한 오류 표시 로직 추가 금지
6) defaultLanguage 비교 후 1개 빼기 같은 임의 제외 로직 금지
7) 직등록, 외부 번역문 즉시등록, 별도 검수탭 복구 금지
8) 유튜브/구글 응답 순화, 해석, 대체 문구 출력 금지
9) 협의되지 않은 문구, 구조, 위치, 버튼명, 흐름 변경 금지
10) 속도를 죽이는 순차전송, 과도한 검증, 파싱 강화 금지
11) Country Code 개수 기준 표시 원칙 임의 변경 금지
12) "사용자를 위해 고쳤다"는 이유로 구조를 뜯어고치는 행위 금지

[허용 범위]
- 배송에 꼭 필요한 최소 연결
- 크롬 팝업 승인 연결
- 유튜브/구글이 실제로 돌려준 원문 오류 표시
- 사용자가 명시 승인한 수정만 반영

한 줄 원칙:
우리는 배송만 한다. 검수하지 않는다. 자체 판단하지 않는다. 거부하지 않는다. 불필요한 오류를 만들지 않는다.
============================================================

[절대 금지 지침 / 다음 담당자 필독]
이 시스템은 배송 시스템이다. 번역기/검수기/품질판단기가 아니다.
- 자체 검수/분석/보정/차단/제외 로직 추가 금지
- defaultLanguage 비교 후 1개 빼기 금지
- 직등록 구조 복구 금지
- 협의 없는 문구/구조/위치 변경 금지
- 유튜브/구글 응답 순화 금지
- "Country Code:" 개수 기준 표시 원칙 임의 변경 금지

팝업 호출 먼저 반영본

1) order.html
- OAuth 클라이언트 ID 입력/구글 승인 시작 구조를
  크롬 팝업 Google 인증 구조로 교체

2) 주문폼 통합 로직
- 크롬 주문폼 로직은 sidepanel.js 기준으로만 유지
- app.js 같은 잔존 파일은 제거

3) oauth_popup.js
- 홈페이지 opener로 access token / channelName 전달

4) manifest.json
- 홈페이지에서 chrome-extension://확장ID/oauth_popup.html 호출 가능하도록
  web_accessible_resources 추가


정리 구조
- sidepanel_intro.html : 크롬 정식 첫페이지(상품소개)
- sidepanel_order.html : 기존 주문페이지(주문 로직 내장)
- sidepanel_order.html 안에 주문 로직을 직접 통합
- app.js / sidepanel.js 는 제거
